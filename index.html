Introducción a las Bases de Datos Relacionales

Una base de datos de tipo SQL se refiere fundamentalmente a una base de datos relacional. Este modelo se basa en los principios postulados por Edgar Frank Codd en 1970, lo que supuso un nuevo paradigma en la gestión de bases de datos.

SELECT * FROM usuarios WHERE pais = 'España';

El modelo relacional organiza los datos en tablas (relaciones) que se conectan entre sí mediante claves, permitiendo una gestión eficiente y consistente de la información.

Definición de SQL

SQL (Structured Query Language, o Lenguaje de Consulta Estructurada) es un lenguaje específico de dominio diseñado para administrar y recuperar información de sistemas de gestión de bases de datos relacionales.

Es un lenguaje declarativo de "alto nivel" y "no procedimental" que permite una alta productividad en la codificación, ya que una sola sentencia puede equivaler a varios programas en un lenguaje de bajo nivel.

Características principales:

  • Lenguaje estandarizado (ANSI/ISO)
  • Independiente del sistema operativo
  • Compatible con múltiples sistemas de gestión de bases de datos
  • Permite interactuar con múltiples registros con una sola orden

Estructura de Datos

Las bases de datos SQL (relacionales) almacenan datos de forma estructurada en tablas, filas y columnas. Los datos están interrelacionados según un modelo que refleja el contenido semántico, y estas interrelaciones se almacenan junto con los datos.

Tablas

Estructuras que contienen los datos organizados en filas y columnas. Cada tabla representa una entidad (ej: usuarios, productos).

Columnas

Atributos o campos que definen las propiedades de los datos almacenados (ej: nombre, edad, email).

Filas

Registros individuales que contienen valores para cada una de las columnas definidas en la tabla.

Normalización

En el diseño de bases de datos relacionales para aplicaciones tradicionales, la normalización es fundamental para reducir la redundancia de datos, optimizar el espacio de almacenamiento y mejorar la calidad de la información.

Manejo de Transacciones

Las bases de datos SQL se utilizan para aplicaciones tradicionales que ejecutan transacciones sobre los datos, lo que implica obtener información, procesarla y almacenar los resultados.

Propiedades ACID

Los sistemas de gestión de bases de datos relacionales suelen garantizar las propiedades ACID:

Propiedad Descripción
Atomicidad Las transacciones se completan totalmente o no se completan en absoluto
Consistencia Las transacciones llevan la base de datos de un estado válido a otro
Aislamiento Las transacciones concurrentes no interfieren entre sí
Durabilidad Los cambios realizados por una transacción persisten incluso ante fallos del sistema
BEGIN TRANSACTION;
UPDATE cuentas SET saldo = saldo - 100 WHERE id = 1;
UPDATE cuentas SET saldo = saldo + 100 WHERE id = 2;
COMMIT;

Componentes del Lenguaje SQL

El alcance de SQL incluye varios sublenguajes especializados:

DDL (Lenguaje de Definición de Datos)

Permite la creación, modificación y eliminación de la estructura de los objetos de la base de datos, como tablas, vistas y procedimientos almacenados.

CREATE TABLE usuarios (
  id INT PRIMARY KEY,
  nombre VARCHAR(50)
);

DML (Lenguaje de Manipulación de Datos)

Incluye comandos para consultar, insertar, actualizar y borrar datos.

INSERT INTO usuarios (id, nombre)
VALUES (1, 'Juan Pérez');

Control de Datos

Permite especificar restricciones de integridad, definir vistas y gestionar los derechos de acceso a los datos.

GRANT SELECT ON usuarios TO usuario_lectura;

Escalabilidad y Aplicaciones

Escalabilidad

A menudo, las bases de datos relacionales tradicionales escalan verticalmente (aumentando la capacidad del hardware de un único servidor), en contraste con las bases de datos NoSQL que suelen escalar horizontalmente.

Aplicaciones y Usos

Las bases de datos SQL son ampliamente utilizadas para la gestión de empresas e instituciones públicas, así como en entornos científicos para almacenar información experimental.

Son la base de muchos sistemas ERP (Enterprise Resource Planning) como Sage, Microsoft Dynamics 365 Business Central, SAP Business One, Epicor ERP e Infor M3, además de plataformas como SharePoint.

Sistemas de Gestión de Bases de Datos Relacionales (SGBDR)

Algunos de los sistemas de gestión de bases de datos relacionales más conocidos que soportan SQL incluyen:

Microsoft SQL Server

Sistema de base de datos desarrollado por Microsoft para entornos empresariales.

MySQL

Sistema de código abierto muy popular para aplicaciones web.

Oracle Database

Sistema robusto y escalable para grandes empresas y aplicaciones críticas.

PostgreSQL

Sistema de código abierto avanzado con amplias capacidades.

MariaDB

Fork de MySQL creado por los desarrolladores originales.

Microsoft Access

Sistema de base de datos para escritorio y pequeñas aplicaciones.

Estandarización y Portabilidad

SQL fue estandarizado por ANSI en 1986 y por ISO en 1987. Sin embargo, a pesar de los estándares, la mayoría de las implementaciones de SQL no son completamente portables entre diferentes sistemas de bases de datos sin ajustes, debido a la complejidad del estándar y las particularidades de cada proveedor.
